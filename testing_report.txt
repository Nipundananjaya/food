1. MODULE A: LOGIN & AUTHENTICATION SYSTEM
--------------------------------------------------------------------------------
Files: login.html, login.php, logout.php, db_connect.php

* TC-LOG-01: Role-Based User Login (Successful)
  - Objective: Verify that staff members can log in using their credentials and 
    role, and are correctly routed to their designated dashboards.
  - Steps:
    1. Open login.html.
    2. Click on the "Admin" role tab.
    3. Input "admin" as username and "admin123" as password.
    4. Click "Sign In".
  - Expected Result: AJAX request is sent to login.php, returning a success JSON, 
    regenerating session id, and redirecting the browser to admin.html.
  - Actual Result: PASS. Session created; redirected successfully.

* TC-LOG-02: Hashed Password Verification (Staff Accounts)
  - Objective: Verify that staff registered via the Admin panel can log in 
    securely using the BCRYPT hashed passwords stored in the database.
  - Steps:
    1. Click on the "Waiter" role tab on the login screen.
    2. Enter a registered waiter's name as username.
    3. Enter their NIC card number (password) as password.
    4. Click "Sign In".
  - Expected Result: The password hash matches in login.php (via password_verify),
    session is initiated, and browser redirects to waiter.html.
  - Actual Result: PASS. Password verified and user logged in.

* TC-LOG-03: Authentication Failures & Validation
  - Objective: Verify that incorrect credentials or empty fields trigger clear 
    error indicators.
  - Steps:
    1. Leave fields empty and click "Sign In". Verify error.
    2. Enter incorrect password for a user. Verify error.
    3. Select "Kitchen" tab but enter an Admin username. Verify error.
  - Expected Result: Appropriate JSON failure responses: "All fields are 
    required", "Incorrect password", or "No kitchen account found with that username".
  - Actual Result: PASS. Clean error labels displayed below form.

* TC-LOG-04: Session Guard & Directory Protection
  - Objective: Ensure unauthorized guests cannot bypass login by typing direct 
    dashboard URLs in the browser.
  - Steps:
    1. Clear browser cookies/sessions.
    2. Try to directly navigate to admin.html, waiter.html, or kitchen.html.
  - Expected Result: Dashboard session checks (e.g. check_session.php) identify 
    no session user_id/role, block page loading, and redirect user to login.html.
  - Actual Result: PASS. Direct navigation is blocked.

* TC-LOG-05: Logout Action
  - Objective: Clear all active credentials on logout.
  - Steps:
    1. Click the "Logout" button on any dashboard.
  - Expected Result: Redirected to logout.php, which runs session_destroy(),
    clears session variables, and redirects back to login.html.
  - Actual Result: PASS. Session cleared immediately.

--------------------------------------------------------------------------------
Files: customer menu.html, menu.html, order_api.php, menu_api.php

* TC-CUST-01: Table Detection via Query Parameters
  - Objective: Verify the customer menu parses the table number from the scanned 
    QR code link automatically.
  - Steps:
    1. Open customer menu.html?table=5 in browser.
  - Expected Result: Interface reads table=5 from query parameters and displays 
    "Table 5" in the header logo.
  - Actual Result: PASS. Correctly loaded table index in header.

* TC-CUST-02: Menu Rendering & Interactive Filtering
  - Objective: Verify menu items load dynamically from the database and can 
    be searched or filtered by category.
  - Steps:
    1. Open the customer menu.
    2. Tap on category badges ("Appetizers", "Beverages").
    3. Type a food item name in the search input box.
  - Expected Result: Category badges isolate respective food items. Search bar 
    filters items in real-time. Unavailable items show "Sold Out" badge.
  - Actual Result: PASS. Rendering is quick, filters update instantly.

* TC-CUST-03: Cart Actions & Calculations
  - Objective: Test adding items to the cart, modifying quantities, and tax calculations.
  - Steps:
    1. Tap "+" on two different food items.
    2. Increase quantity of one item.
    3. Verify cart drawer values.
  - Expected Result: Subtotal updates correctly. Service charge & tax (10%) is 
    calculated, and Grand Total matches the formula (Subtotal + (Subtotal * 10%)).
  - Actual Result: PASS. Values calculated accurately.

* TC-CUST-04: Checkout & Transaction Integrity
  - Objective: Ensure order is written to DB using a transaction so no partial 
    or orphaned order records are created.
  - Steps:
    1. Tap "Place Order" button in the cart.
  - Expected Result: JSON payload is sent to order_api.php. It inserts a row 
    into 'orders', loops through items, retrieves actual database prices, 
    adds item records, commits the transaction, and returns the order_id.
  - Actual Result: PASS. Order recorded, database table state matches.

* TC-CUST-05: Real-Time Order Tracking
  - Objective: Track placed order statuses without manual page reloads.
  - Steps:
    1. Complete checkout to get order tracker.
    2. Update status of the order in the Kitchen or Waiter panel.
  - Expected Result: The tracking badge on the customer menu updates state from 
    "Pending" -> "Preparing" -> "Ready to Serve" -> "Served" using polling.
  - Actual Result: PASS. Tracker badge reflects exact database status changes.
